# AI生成花材プロンプト集（全29種・季節別）

対象: 花材調達管理シート 全項目（元28種＋牡丹の追加項目）
版: v2.5（2026-07-15）— ①②区分の暫定素材化を開始（梅（残春）・海棠・蘭・桃の4種を生成・登録）
旧版: v2.4（2026-07-15）— 牡丹・石榴花・桂花をMLセグメンテーション（rembg/isnet）で再切り抜き（最終版）
旧版: v2.3（2026-07-15）— 仏手柑は考証で不採用。牡丹・石榴花・桂花は指定候補で高精度に再切り抜き
旧版: v2.2（2026-07-15）— ③区分5種を全てAI生成・採用（生成実績を各項に記録）
旧版: v2.1（2026-07-15）— v2.0再構成時に脱落していた③区分5種（牡丹・石榴花・桂花・蝋梅・仏手柑の盛物）を補筆
旧版: v2.0（2026-07-12）— 全種プロンプトを整備し、季節別に再構成

## 使い方

1. 各プロンプトを、お使いのAI画像生成ツール（Google AI Studio / Imagen、その他）にそのまま貼り付ける
2. 生成後、**花材撮影仕様書§6**の基準に従い、学芸員（Junt）による考証チェックを行う
3. 採用したら背景を透過処理し、留め（茎の切り口）の位置をアンカー座標として記録する
4. `素材調達管理シート.xlsx` の該当行に、状態・使用ツール名・生成日を記録する

**重要**：本プロンプト集は全29種を対象に用意していますが、調達区分（①②③）は花材ごとに異なります。①（華道家連携撮影）・②（ストック）を優先する花材についても、AI生成を**下書き・比較用の参考素材**として使うことは可能ですが、最終採用は`素材調達管理シート.xlsx`の調達区分に従ってください。①②で調達予定の花材にAI生成品をそのまま採用する場合は、必ず学芸員の考証チェックを経てください。

各プロンプトは共通して以下を含みます：花器を含めない／背景は無地で切り抜き前提／影を作らない均一光／茎の切り口（留め）が下端に見えること／実物写真としての質感（イラスト調にしない）。盆栽・盛物は器・盤を含めず、根・株・果実等の構成要素のみを写します。

## 梅（残春）— 文人花・草・narrow口径【調達区分①】

**様式メモ**：ウメ（Prunus mume）の晩春、名残の数輪。ねじれた古木質の枝に、5弁の白〜淡紅色の花をまばらに残す。文人花・草の様式では、満開ではなく散り際の風情を一枝に凝縮する。

```
中国の文人花（瓶史）の様式で生けた梅（残春）の一枝。実物の植物写真、マクロ撮影風。
ねじれ、節くれ立った古木質の暗褐色の枝に、5弁の白〜淡いピンクの花を2〜3輪だけ残している。
満開ではなく、名残の風情。葉はまだ出ていないか、ごくわずか。
花器・鉢・器は一切写り込ませない。背景は無地の中間グレー、均一な柔らかい光で影を作らない。
枝の下端は剪定鋏で切ったような清潔な切り口。縦長構図、正面よりやや見下ろす角度。
写実的な植物写真、イラストや水彩調にはしない。
```

**ネガティブ指定**：満開の花、桃や桜との混同、花瓶、鉢、CGらしい過剰な光沢

**生成実績**：2026-07-15、Canva AIで生成・**暫定採用**（F001、spring-bunjin-ume-zanshun.webp、`--ml`）。候補1を採用。本調達区分は①のため、華道家撮影素材が届き次第差し替える。

## 仏手柑の盛物（ぶっしゅかんのもりもの）— 盛物・flat口径【調達区分③】

**様式メモ**：ブッシュカン（Citrus medica var. sarcodactylis）は果実の先が仏の手の指のように分かれる香酸柑橘。鮮黄色でつやのある果皮。清供（文房の供物）として果実そのものを盤に盛る伝統がある。器・盤は含めない。

```
仏手柑（ぶっしゅかん）の果実の盛物構成。実物の植物写真、マクロ撮影風。
果実の先端が仏の手の指のように細長く分かれた、鮮やかな黄色でつやのある柑橘の果実を
1〜2個並べて配置する。表面は柑橘らしい細かい凹凸のある果皮。濃緑色の葉を1〜2枚添えてもよい。
器・盤・鉢は一切写り込ませない。背景は無地の中間グレー、均一な柔らかい光で影を作らない。
横長〜縦長構図、正面よりやや見下ろす角度。
写実的な植物写真、イラストや水彩調にはしない。
```

**ネガティブ指定**：レモン・ゆず（指状に分かれない）との混同、器・盤に盛られた状態、CGらしい過剰な光沢

**生成実績**：2026-07-15、Canva AIで生成したが**学芸員の考証で不採用**（アプリ未登録・F028欠番）。盛物は現行サンプル作品に適合器が無いこと、及び画像の完成度から見送り。再挑戦時は本プロンプトを流用可。

## 生成がうまくいかない場合の調整ヒント

- 花器が写り込んでしまう場合：プロンプト末尾に「花瓶や鉢、盤、石は絶対に描写しないこと」を追加、または生成後にツール側の「除去（inpaint）」機能で消してから採用
- 影が強く出る場合：「影のない、拡散光での撮影」という指定を強調
- イラスト調になってしまう場合：「実際のカメラで撮影した写真、非イラスト、非CG」を明記
- 類似種と混同される場合：ネガティブ指定に挙げた「〜との混同」の項目を強調し、識別点（花弁の付き方、葉の形状など）を本文にも追記する
