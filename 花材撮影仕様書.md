# 花材撮影仕様書 — 花をいける AR花いけアプリ

対象: ①華道家連携撮影 ②AI生成素材の体裁統一の基準にも準用
版: v1.0（2026-07-12）

---

## 1. 目的

来館者のスマートフォン画面上で、実際の展示陶磁器の口に「生けた花」が重なって見えるための素材を作る。
**花器は写真に写り込ませない**。花器は展示室の実物が担い、写真は花の部分だけを提供する。

## 2. 仕上がり仕様（共通・厳守）

| 項目 | 仕様 |
|---|---|
| 出力形式 | WebP（透過）。撮影・切り抜き元はRAWまたはPNG可 |
| 解像度 | 縦1200px基準（長辺上限1600px） |
| 1点あたり容量 | 150〜250KB目安（全体8MB以内） |
| 背景 | 完全透過。切り抜き境界に縁の色残り（フリンジ）が出ないこと |
| 下端の終端 | **「留め」**（茎の束・剣山への刺し口）で切れていること。花器の縁や剣山自体は写り込ませない |
| ファイル命名 | `{季節}-{様式}-{花材名}.webp` 例: `winter-bunjin-robai.webp` |

## 3. 撮影時の手順（①連携撮影用）

1. **無地背景**（グレー、明度中間）を用意する。白背景は花の白い部分（梅・水仙の花弁など）との境界が消えやすいため避ける
2. **順光**で撮影し、強い影を作らない。影があると透過処理時に不自然な暗部が残る
3. 花材は**実際に花器（または花器に近い形状の代用容器）に生けた状態**で撮影する。これにより「留め」の自然な角度・広がりが記録される
4. カメラは花の**正面やや上**（見下ろし気味）から。来館者がスマホを構える角度に近い
5. 同一花材について、可能なら**2〜3構図**（真正面・やや斜め）を撮っておくと、後日「行」「草」など複数様式への転用がしやすい
6. 撮影後、花器を写さず**花の部分だけ**をトリミングする前提でよいので、撮影時に花器全体が入っていても構わない（切り抜きは後工程で行う）

## 4. 切り抜き作業

1. Photoshop等の被写体選択機能、またはremove.bg等のオンラインツールで背景を透過
2. **花器を含めず**、留め（茎の束）の直下で切る
3. 切り抜き後、フリンジ（縁の色残り）が無いか200%表示で確認
4. 「アンカー点」（留めの位置、画像内の相対座標 x, y ともに0〜1）を計測し記録する。目安は `{x: 0.5, y: 0.96}` 付近（下端中央）

## 5. 様式ごとの構図メモ（学芸的な留意点）

| 様式 | 構図の要点 |
|---|---|
| 真 | 中心軸に対して左右対称に近い、格の高い端正な構成。枝ぶりの矯めが少ない状態を選ぶ |
| 行 | 真と草の中間。やや傾き・非対称を許容 |
| 草 | 一輪・一枝の投げ入れに近い、崩した構成 |
| 文人花 | 『瓶史』考証に基づく花材（梅・蘭・竹・菊などの四君子、水仙等）。細く伸びやかな一枝が基本 |
| 盆栽・盛物 | 器の口が広い場合の構成。複数の実・葉を含む場合がある |

## 6. AI生成素材（③）を使う場合の追加基準

AI生成であっても、**上記2〜4の仕上がり仕様は同一**とする。加えて以下を満たすこと。

1. 生成前に、様式・花材の考証（本仕様書§5および引き継ぎ一式の花材リスト）に沿ったプロンプトを用意する
2. 生成後、**学芸員（Junt）による考証チェック**を経てから採用する（花弁の枚数、葉の形状、開花時期の齟齬がないか等）
3. 生成ツールの利用規約で、**商用・公開Webアプリでの利用が許可**されているか確認する
4. 採用した生成素材は、素材調達管理シートに「AI生成」と明記し、使用したツール名・生成日を記録する

## 7. ストックフォト（②）を使う場合の追加基準

1. 様式に厳密に沿った生花のストックは少ないため、**汎用的な一輪・一枝もの**（桃・南天・竜胆等、様式を問わない花材）を優先して探す
2. 購入前に、ライセンス条項が **「公開Webアプリケーションでの改変・再配布（切り抜き加工を含む）」を許可しているか** を必ず確認する（多くのストックサービスは「印刷物のみ」「改変禁止」等の制限がある）
3. 背景が単色・無地に近い商品写真を選ぶと、切り抜き作業が容易になる

## 8. クレジット表記

調達区分にかかわらず、以下をアプリ内（メーカーモードのQR発行時、または将来のクレジット画面）に記録できるようにする。

- ①連携撮影: 協力した流派・華道家名
- ②ストック: 素材提供元サービス名（ライセンス規約で表記義務がある場合）
- ③AI生成: 使用ツール名（内部記録。利用規約次第で画面表示も検討）

## 9. チェックリスト（納品時）

- [ ] 花器が写り込んでいない
- [ ] 背景が完全透過、フリンジなし
- [ ] 留めの位置がアンカー座標として記録されている
- [ ] WebP形式・容量が仕様内
- [ ] ファイル名が命名規則に沿っている
- [ ] 調達区分（①②③）とクレジット情報が記録されている
- [ ] （AI生成の場合）学芸員の考証チェック済み
- [ ] （ストックの場合）ライセンス条項を確認済み
